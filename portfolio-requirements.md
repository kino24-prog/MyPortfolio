## 9. デザイン要件

| 項目 | 要件 |

## 10. 機能要件

| 機能 | 必須 | 内容 |
|---|---|---|
| 外部リンク | 必須 | X、Instagram、GitHub、itch.ioへ遷移できる |
| レスポンシブ対応 | 必須 | スマホ、タブレット、PCで破綻しない |
| 作品一覧 | 必須 | 作品カードを一覧表示する |
| 作品詳細 | 任意 | 工夫した点、苦労した点を表示する |
| スクリーンショット表示 | 必須 | 作品カードに表示する |
| お問い合わせフォーム | 不要 | 初期は実装しない。メールリンクで対応する |

## 11. 非機能要件

| 項目 | 要件 |
|---|---|
| 表示速度 | 画像を軽量化し、初回表示を重くしない |
| 保守性 | 作品データを配列などで管理し、追加しやすくする |
| アクセシビリティ | リンクの意味がわかるテキスト、十分なコントラストを確保 |
| SEO | title、description、OGP設定を行う |
| セキュリティ | 外部リンクには必要に応じて `rel="noopener noreferrer"` を設定 |

| 項目 | 今回の実装 |
|---|---|
| フレームワーク | なし |

| スタイリング | CSS |

初期実装では、学習コストと編集しやすさを優先して、依存関係なしの静的サイトとして作成する。
後からNext.jsやReactへ移行する場合でも、作品データを `data.js` に分離しているため移し替えやすい。

## 13. コンテンツ管理方針

作品情報は `data.js` の配列として管理する。

```ts
type Project = {
  title: string;
  description: string;
  url: string;
  period: string;
  teamSize: number;
  role: string[];
  technologies: string[];
  jam: string;
  highlights?: string[];
  challenges?: string[];
  image?: string;
};
```

プロフィール情報も `data.js` のオブジェクトで管理する。

```ts
type Profile = {
  name: string;
  title: string;
  bio: string;
  links: {
    x: string;
    instagram: string;
    github: string;
    email: string;
  };
};
```

## 14.1 作成ファイル

| ファイル | 役割 | 編集頻度 |
|---|---|---|
| `index.html` | ページ構造 | 低 |
| `styles.css` | 見た目、レスポンシブ対応 | 中 |

## 16. 実装優先順位

| 順位 | 作業 | 理由 |
|---|---|---|
| 1 | 1ページ構成を作る | まず公開できる形にする |
| 2 | Heroで名前と志望職種を明確に出す | 最初に何者かを伝えるため |
| 3 | 作品カードを実装する | 実績がサイトの中心だから |
| 4 | SNS・GitHub導線を置く | 訪問者が次に動きやすくなる |
| 5 | レスポンシブ対応する | スマホ閲覧が多いため |
| 6 | SEO・OGPを設定する | SNS共有時に見栄えを整える |
| 7 | スクリーンショットを追加する | 視覚的に作品を伝える |
